Diagram Components

  • Data Source: CSV file with 57,000 flight records
  • Orchestration: Apache Airflow with 4 sequential DAG tasks
  • Staging: MySQL with raw and validated data tables
  • Analytics: PostgreSQL with medallion architecture (Bronze/Silver/Gold)
  • Transformations: DBT with 9 models and 2 SCD Type 2 snapshots

Architecture Layers

  • Bronze Layer: 57,000 validated flight records
  • Silver Layer: Cleaned and standardized data with derived columns
  • Gold Layer: 6 KPI tables for business analysis

Pipeline Tasks

  1. Task 1 (Load CSV): Read 57,000 records and load to MySQL (~8s)
  2. Task 2 (Validate): Validate data and set quality flags (~10s)
  3. Task 3 (Transfer): Move valid records to PostgreSQL (~12s)
  4. Task 4 (DBT Run): Execute transformations and snapshots (~4s)
